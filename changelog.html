Cadence

Releases follow the calendar, quarterly minor versions in February, May, August, and November. Patch versions can land any time for broken-link fixes, accessibility tweaks, or new citations. Major versions happen once a year on the back of a full content audit.

Version scheme: vYYYY.MM. The version of the page you're currently looking at is in every footer.

v2026.06 · LMS distribution + EU-language shells

v2026.06

Released 2026-06-03

What's new

  • SCORM 1.2 package, IMS Content Aggregation Model v1.2 manifest, single-SCO, 80% mastery score, 2-hour session cap. API discovery walks parent opener chains; cmi.suspend_data persists per-module completion across sessions. Builder script under scripts/build-scorm.py (stdlib zip, zip-slip safe, --locale en|fr|de|all). Deployable on Moodle, Cornerstone, Workday Learning, SCORM Cloud.
  • xAPI (Tin Can) adapter, HTTPS-gated POST to any conformant LRS. Anonymous-actor default, no auth-in-URL, credentials: 'omit'. IRI namespace lives at https://attest-ai.com/xapi/v2/. Spec + LRS config template under .audit/integrations/.
  • FR and DE language shells, full v2 site rendered in French and German (/v2/fr/ and /v2/de/). Banner on every page declares machine-translation status and links to canonical EN. modules.json extended with locale dictionaries; v2.js auto-detects locale from path.
  • SEO + AEO metadata pass, JSON-LD WebSite / Course / LearningResource / BreadcrumbList + canonical + hreflang (en/fr/de/x-default) + OpenGraph + Twitter card across all 39 v2 pages. Repeatable splicer: scripts/inject-v2-seo.py (idempotent, marker-based).

Procurement gates closed this release

  • Gate 4, SCORM 1.2 + xAPI export wrapper.
  • Gate 5, FR + DE translations (shells; native-speaker review pending).
  • Gate 13, JSON-LD + hreflang + sitemap (v2 deferred until noindex lifted).
  • Gate 14, OpenGraph + Twitter card meta (v2).

Open flags to resolve before next release

  • Native-speaker review of FR + DE shells before procurement deployment.
  • Custom domain (attest-ai.com) purchase + DNS, recipe filed at .audit/legal/domain-procurement-2026-06-03.md.
  • OMB M-25-21 URL canonical form (carried over from v2026.05).
v2026.05 · Initial release

v2026.05

Released 2026-05-19

What's new

  • 11 universal modules (M1–M12) covering: why this course exists, what AI tools do with data, the never-paste list, picking the right tool, verifying outputs, AI-powered scams, bias and fairness, copyright and IP, logging, incident response, the 60-second checklist, the standards.
  • 2 Relevant Roles: Manager and DPO / compliance lead.
  • 2 ship-ready templates: Acceptable Use Policy, AI vendor diligence questionnaire.
  • Phase 2 Standards Map, 6 standards, 57 clause-to-module rows, 88 cross-link pills.
  • Citations bibliography, every claim mapped to its primary source with a confidence flag.
  • Cinematic design system: aurora + drifting orbs + parallax + scroll reveals + reading progress + press feedback. Full prefers-reduced-motion fallback. Clean print path.

Standards baseline

  • EU AI Act, Regulation (EU) 2024/1689 (Art 4 in force 2 Feb 2025; phased through 2026–2027).
  • GDPR, Regulation (EU) 2016/679.
  • ISO/IEC 42001:2023, AI management system.
  • ISO/IEC 27001:2022, Information security management (93 Annex A controls).
  • NIST AI RMF 1.0 + GAI Profile (NIST AI 600-1).
  • OWASP Top 10 for LLM Applications (2025 edition).

Resolved flags (mid-release)

  • EU AI Act Art 22 cross-reference numbering on the standards-map page. Resolved 2026-05-19, Art 22 is "Authorised Representatives", not auto-decisions. Row removed from standards-map.
  • ISO/IEC 42001 Annex A sub-clause numbering for A.3.2 and A.6.2.5. Resolved 2026-05-19, A.3.2 = "AI Roles and Responsibilities" (not "AI policy"); A.6.2.5 = "AI-System Deployment" (not incidents); M10 reference now points to A.6.2.6 + A.9 for incident handling.

Open flags to resolve before next release

  • OMB M-25-21 URL canonical form (still open, verify in next quarterly).

Why this page exists

An auditor reviewing AI-related training will ask: "what version of this course did the team take, and when was it last updated against current regulation?" This page is the answer. The version a learner sees is dated in the footer of every module; the contents of each released version is here.

If the most recent release is more than a quarter old, treat the content as pending refresh and verify any single-clause claim before relying on it. See the citations bibliography for current confidence levels.
