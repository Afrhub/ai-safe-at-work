AI Use Case Register

Record every way your organisation uses AI: what problem it solves, who uses it, what data it touches, whether a DPIA or FRIA has been done. Aligns with EU AI Act Articles 4, 27; GDPR Article 35; ISO/IEC 42001 Clause 6.2.

Maintain in a shared spreadsheet. Update whenever a new AI use case is introduced or an existing one changes.
Purpose and scope

Why we keep a use case register

A Use Case Register records what we actually do with AI — not the tools we have, but the jobs we use them for. It:

  • Traces data flows: What data does this AI use case ingest, process, and output?
  • Triggers impact assessment: Does this use case trigger a GDPR DPIA or EU AI Act FRIA requirement?
  • Ensures governance oversight: Every use case has an owner and a risk level.
  • Provides audit evidence: Shows regulators that we've thought about each use of AI.

Scope

Record every distinct way AI is used, including:

  • Individual staff members using ChatGPT to draft documents.
  • Support team using AI to summarize customer emails.
  • Hiring team using AI to screen CVs.
  • Finance team using AI to categorize invoices.
  • Any other task where the outcome of AI processing affects decision-making, customer experience, or business decisions.

Register columns explained

Use case ID and name

Give each use case a unique ID (UC-001, UC-002, etc.) and a descriptive name. Examples: "Customer email summarization", "Invoice categorization", "CV screening for data analyst role".

Purpose

What problem does this use case solve? One sentence. Example: "Reduce time support staff spend reading and summarizing customer emails."

AI tool(s) used

Which tool(s) power this use case? (Reference the approved AI Tool Register.) Example: "ChatGPT Enterprise".

Input data categories

What data does this use case ingest? Choose from: public data, internal non-sensitive, confidential business, personal data (general), special-category data. List all that apply.

Data volume / retention

How much data flows through? Example: "500 emails/month, retained by staff in summaries (not in ChatGPT)."

Affected parties

Whose data is processed? Example: "Customers (email authors)". This determines whether GDPR applies and what rights apply.

Use case owner

Named person responsible for this use case. They ensure the use case complies with policy, monitor for issues, and report status.

GDPR impact assessment required?

Yes / No / Pending. A DPIA (Data Protection Impact Assessment per GDPR Article 35) is required if:

  • You process special-category data (health, ethnicity, religion, etc.).
  • You make automated decisions about people (hiring, credit, eligibility).
  • The processing involves large-scale personal data.

EU AI Act impact assessment required?

Yes / No / Pending. An FRIA (Fundamental Rights Impact Assessment per EU AI Act Article 27) is required if the AI system could impact fundamental rights (privacy, fair trial, non-discrimination, etc.). Most customer-facing or decision-affecting use cases trigger FRIA.

Assessment status and location

If a DPIA or FRIA is required, where does it live? Link to the document location. Example: "DPIA-001 on shared drive / AI-Assessments/Customer-Email-Summary-DPIA.pdf".

Risk rating

High / Medium / Low — based on data sensitivity and impact. A use case that processes special-category data and makes automated decisions is High. Use your AI Risk Register to inform this.

Compliance status

Active / Paused / Discontinued. Is this use case running, on hold, or ended?

Notes

Any other relevant detail. Example: "Under review pending DPIA completion."

How to establish and maintain the register

  1. Discovery: Interview managers, team leads, and department heads. Ask: "What are you using AI for?" Include shadow AI (unapproved uses).
  2. Record: For each use case, complete the columns above. Identify which ones require DPIA / FRIA.
  3. Assess: For High-risk use cases or those with personal data, trigger DPIA / FRIA immediately. Assign an owner to complete it within 30 days.
  4. Update monthly: Review the register with department heads. Any new use cases? Any discontinuations?
  5. Link to risk register: Each use case on this register should have a corresponding entry (or entries) in the AI Risk Register if risks are identified.

Standards referenced

  • EU AI Act 2024/1689 — Article 4 (literacy), Article 27 (deployer obligations to assess and monitor).
  • GDPR 2016/679 — Article 35 (Data Protection Impact Assessment requirement).
  • ISO/IEC 42001:2023 — Clause 6.2 (determining requirements for AI systems).

Getting started
  1. Set up a shared spreadsheet with columns: Use Case ID, Name, Purpose, Tool, Input Data, Owner, DPIA?, FRIA?, Risk, Status.
  2. Run interviews with each department: "What are you using AI for?" List high-value and shadow-AI uses.
  3. For each use case, record the columns. Highlight any that involve personal data or automated decisions — these need DPIA / FRIA.
  4. Assign owners and trigger assessments for High-risk use cases.
  5. Review monthly. Update immediately if a new use case begins or vendor terms change.
