Accessibility statement · v2026.06

Accessibility.

We target WCAG 2.2 Level AA on every page. The site is built static and text-first, with full prefers-reduced-motion support and a clean print path. Something broken for you? The complaints route is on this page.

Conformance status

Self-assessed as partially conformant with WCAG 2.2 Level AA as of 2026-06-03. First-pass internal audit completed 2026-06-03, three Level A / AA fails fixed (skip-to-content link, focus-visible styles, body-text contrast ratio). Eight items deferred to a manual + assistive-technology pass scheduled for 2026-09. A third-party audit will follow once the next-quarter manual pass is closed.

Audit record: wcag-audit-2026-06-03.md on GitHub.

What works well

  • Semantic HTML throughout, proper heading hierarchy, landmark elements, lists where lists are needed.
  • Sufficient colour contrast on body text, headings, and links against the dark background.
  • Keyboard-operable: every interactive element can be reached and activated without a mouse.
  • No autoplay video, no auto-advancing carousels, no time limits.
  • Full prefers-reduced-motion: reduce respect, the cinematic background and scroll reveals are switched off automatically for users with that preference.
  • Print stylesheet, modules can be printed cleanly; the 60-second checklist is designed for print.
  • No login, no forms, no captcha to navigate.
  • Mobile-first responsive layout; supports orientation changes and pinch zoom.
  • No tracking cookies, no third-party scripts beyond Google Fonts.

Resolved 2026-06-03 (first-pass audit)

  • Skip-to-content link, added on every page. Press Tab from the page top to focus it; Enter jumps to the main content landmark.
  • Focus styles, custom high-contrast amber outline on every interactive element via :focus-visible. Mouse-clicks don't trigger it; keyboard users always see it.
  • Body-text contrast, the --text3 caption / hint colour was bumped to deliver ≥ 6.7:1 contrast against the dark background (WCAG AA needs 4.5:1 for normal text).

Known issues (deferred to next-quarter manual + AT pass)

  • Manual screen-reader pass, internal NVDA + VoiceOver pass scheduled for 2026-09. A third-party AT audit follows once internal findings are closed.
  • Cinematic background, the aurora gradient layer adds subtle visual motion. Users on prefers-reduced-motion: reduce get a static version; we may add a manual toggle for users who don't set the OS preference but find it distracting.
  • Touch target sizing, WCAG 2.2 SC 2.5.5 (Target Size Minimum) needs per-viewport measurement, not static review. Re-evaluated next quarter.
  • Focus-not-obscured, WCAG 2.2 SC 2.4.11 against the sticky topbar. Needs interactive scroll testing; deferred to next pass.
  • Some module text uses long sentences in places where the meaning depends on nuance. We accept this trade-off, readability for an adult professional audience over Hemingway-short sentences, but acknowledge it raises cognitive load for some readers.
  • No alternative formats currently offered (audio, summarised plain-English version). On the roadmap.

Assistive technology compatibility

Designed and tested-by-developer with screen readers in mind. A third-party screen-reader audit is part of the planned external audit. If you use NVDA, JAWS, VoiceOver, or TalkBack and encounter a barrier, please report it via the complaints route.

How to report a barrier

If something on this site doesn't work for your assistive technology, your device, or your situation, please tell us. See the complaints procedure. We respond within five working days.

Enforcement

EU residents have the right to raise an accessibility complaint with the relevant national enforcement body. We will provide contact details on request, and we aim to resolve issues directly first.

Preparation of this statement

This statement was prepared on 2026-05-19. It was last reviewed on 2026-06-03 (first-pass internal audit closed; conformance and known-issues sections updated). It will be reviewed every quarter alongside the standards refresh, and after any material change to the site.
